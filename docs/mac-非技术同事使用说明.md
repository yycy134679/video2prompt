# video2prompt mac 使用说明

这份文档面向不写代码的同事。

你不需要提前安装：

- `git`
- `VSCode`
- Docker

你只需要：

- 一台 mac
- Chrome 浏览器
- Python 3.11 或更高版本
- 火山方舟账号信息
- 可以登录抖音网页的账号

## 一、你需要准备什么

开始前请先准备好这几样内容：

1. `VOLCENGINE_API_KEY` 或 `ARK_API_KEY`
2. 火山方舟的 `endpoint_id`
3. Chrome 浏览器
4. 抖音网页登录状态

如果你只是做主流程，不需要提前安装：

- `ffmpeg`
- `ffprobe`
- Gemini Key

这些都属于进阶功能。

## 二、如何下载和解压

1. 从内部共享位置或 GitHub 下载项目 ZIP 包
2. 双击 ZIP 解压
3. 打开解压后的项目文件夹

你后面主要会用到这 3 个文件：

- `scripts/mac/安装.command`
- `scripts/mac/启动.command`
- `scripts/mac/停止.command`

## 三、第一次安装

### 1. 先确认有没有 Python

打开“终端”后输入：

```bash
python3 --version
```

如果看到版本号，并且是 `3.11` 或更高版本，就可以继续。

如果提示没有 `python3`，或者版本低于 `3.11`：

1. 打开 [Python for macOS](https://www.python.org/downloads/macos/)
2. 下载最新的 Python 3.11+
3. 安装完成后，重新打开终端再执行一次 `python3 --version`
Python 3.11下载链接（点击可直接下载安装包）：https://www.python.org/ftp/python/3.11.9/python-3.11.9-macos11.pkg

说明：

- 不需要安装 Homebrew
- 不需要安装 Anaconda
- 不需要配置 `git`

### 2. 双击运行安装脚本

在 Finder 中双击：

- `scripts/mac/安装.command`

安装脚本会自动完成这些事情：

- 检查网络
- 检查 Python 版本
- 创建本项目运行环境
- 下载并安装本地解析服务
- 安装内置兜底解析能力
- 写入解析服务默认配置

### 3. 如果 mac 提示“未打开”“Apple 无法验证”

第一次运行脚本时，macOS 可能会拦截。

如果你看到类似下面这种提示：

- “未打开 `安装.command`”
- “Apple 无法验证 `安装.command` 是否包含可能危害 Mac 安全或泄漏隐私的恶意软件”

请按下面处理：

1. 在弹窗里点击“完成”，不要点“移到废纸篓”
2. 回到 Finder，右键 `安装.command`
3. 选择“打开”
4. 如果仍被阻止，打开：
   - “系统设置” > “隐私与安全性”
5. 在页面下方找到被阻止的脚本
6. 点击“仍要打开”或允许
7. 再次运行脚本

## 四、启动工具

安装完成后，双击：

- `scripts/mac/启动.command`

它会自动：

- 启动本地解析服务
- 启动网页工具
- 自动打开浏览器

如果浏览器没有自动打开，可以手动访问：

- [http://127.0.0.1:8501](http://127.0.0.1:8501)

停止工具时，双击：

- `scripts/mac/停止.command`

## 五、第一次打开页面要做什么

页面打开后，请先进入：

- `首次设置 / 环境检查`

先完成主流程配置，再去执行任务。

### 1. 填写火山方舟信息

在页面里填写：

- `VOLCENGINE_API_KEY` 或 `ARK_API_KEY`
- `endpoint_id`

说明：

- 这是主流程默认使用的模型服务商
- 只填写其中一个火山 API Key 就可以

### 2. 粘贴抖音 Cookie

在页面里找到：

- `抖音 Cookie`

把你从 Chrome 复制出来的整段 Cookie 粘贴进去，然后点击“保存设置”。

## 六、如何从 Chrome 获取抖音 Cookie

1. 打开 Chrome
2. 登录抖音网页
3. 在抖音页面按 `Option + Command + I`
4. 打开开发者工具后，切到 `Network`
5. 刷新页面
6. 随便点开一条请求
7. 在右侧或下方找到 `Request Headers`
8. 找到 `Cookie`
9. 复制完整内容
10. 回到工具页面，把整段内容粘贴到 `抖音 Cookie`

建议：

- 尽量使用你平时正常登录、能正常浏览抖音的账号
- Cookie 失效后，重新复制并保存一次即可

## 七、如何检查配置是否成功

仍然在“首次设置 / 环境检查”页面里：

1. 先点 `检查服务状态`
2. 再用一条抖音链接点 `测试解析`

如果测试解析成功，说明：

- 本地解析服务正常
- Cookie 基本可用
- 可以进入正式任务

补充说明：

- 系统会优先使用内置本地解析服务
- 如果个别抖音链接因为平台风控导致主解析失败，工具会自动尝试第二种兜底解析方式
- 这一步不需要你手动切换或额外安装别的工具

## 八、如何执行任务

进入：

- `任务执行`

### 1. 选择运行模式

常用模式有：

- `视频复刻提示词`
- `按类目分析`
- `视频时长判断`

对大多数同事来说，先使用默认模式即可。

### 2. 输入任务内容

常规模式下需要两列：

- `pid 列表`
- `抖音/TikTok 链接列表`

要求：

- 一行 pid 对应一行链接
- 行数要对齐
- 空白行会自动忽略

### 3. 点击开始执行

点击：

- `开始执行`

执行过程中你可以看到：

- 当前状态
- 重试次数
- 是否命中缓存
- 错误原因

如需中止，点击：

- `停止`

## 九、如何导出结果

任务完成后，页面下方会出现导出区域。

你可以按当前模式下载：

- Excel
- Markdown ZIP
- 时长模式双 Excel

如果浏览器下载失败，也可以直接去本地目录找文件：

- `exports/`

## 十、常见问题

### 1. 页面能打开，但点开始执行报配置错误

优先回到“首次设置 / 环境检查”检查：

- 火山 API Key 是否已填
- `endpoint_id` 是否已填
- 抖音 Cookie 是否已填

### 2. 提示“解析服务不可用”

处理顺序：

1. 先双击 `启动.command`
2. 回页面点 `检查服务状态`
3. 如果还不行，先双击 `停止.command`
4. 再重新双击 `启动.command`

### 3. 测试解析失败

常见原因：

- 抖音 Cookie 过期
- 当前账号没有正常登录
- 当前网络无法正常访问抖音

通常重新登录抖音网页，再重新复制 Cookie 即可。

### 4. 提示没有 `ffprobe`

这只会影响：

- `视频时长判断`

如果你只是跑普通的视频分析，可以先忽略。

如果需要时长判断，再安装 `ffmpeg` 即可。

### 5. 安装脚本运行失败

优先检查：

- Python 是否为 3.11+
- 网络是否能访问 GitHub、PyPI、Python 官网
- macOS 是否拦截了脚本执行

## 十一、进阶使用

### 1. Gemini 配置

如果你要改用 Gemini：

1. 回到“首次设置 / 环境检查”
2. 在“进阶设置”里填写 `GEMINI_API_KEY`
3. 把默认模型服务商改成 `gemini`
4. 保存设置

### 2. TikTok Cookie

如果你的任务需要更稳定地处理 TikTok，可以在“进阶设置”中填写：

- `TikTok Cookie`

### 3. 时长判断模式

如果你要使用：

- `视频时长判断`

你需要额外安装 `ffmpeg`，因为这个模式依赖 `ffprobe`。

### 4. 受管解析服务存放位置

如果维护者需要排查问题，受管解析服务默认在：

- `.managed/douyin_tiktok_download_api`

常见位置包括：

- 源码目录：`.managed/douyin_tiktok_download_api/source`
- 日志目录：`.managed/douyin_tiktok_download_api/runtime`

## 十二、建议的日常使用顺序

1. 双击 `启动.command`
2. 打开“首次设置 / 环境检查”确认状态
3. 必要时更新 Cookie
4. 进入“任务执行”
5. 粘贴 `pid + 链接`
6. 执行并导出结果
7. 完成后双击 `停止.command`
